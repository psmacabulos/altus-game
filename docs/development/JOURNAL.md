# Development Journal

## Purpose

This journal is my chronological record of building the Altus Game project. Rather than only tracking what was completed, I also want to capture what I learned, the challenges I encountered, and the reasoning behind the decisions I made.

The goal is to document the development process from the very first commit through to the finished game, so I can look back at how the project evolved over time.

# Entry 001

**Date:** 2026-06-26

## Today's Progress

- Created the Altus Game repository.
- Planned the project documentation structure.
- Defined the initial engineering principles.
- Chose to build the game as a standalone Phaser project.
- Agreed on using a public API (`createGame()` and `triggerRep()`) to keep the game independent from the frontend.
- Decided to setup the development environment first before coding.

Good architecture starts before writing code. Spending time defining the project structure and documenting key decisions should make development easier as the project grows.
